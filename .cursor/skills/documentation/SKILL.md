# Documentation Generator

## Purpose

Produce clear, accurate documentation from source code to improve developer onboarding and API discoverability.

## Instructions

When generating documentation:

1. **API Reference** — Generate endpoint docs with method, path, parameters, request/response schemas, and examples
2. **Function Docstrings** — Write Google-style docstrings with Args, Returns, Raises, and Examples sections
3. **Module Overview** — Create a top-of-file docstring explaining the module's purpose and key classes
4. **README Sections** — Generate Getting Started, Configuration, and Usage sections
5. **Changelog** — Summarize changes between versions using Keep a Changelog format

## Formatting

- Use Markdown for all documentation files
- Include code examples with syntax highlighting
- Link to related modules and external references
- Keep line length under 100 characters for readability
